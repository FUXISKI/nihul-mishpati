סנכרון ענן — שרת כספת (vault-server)
=====================================

מה השרת עושה
-------------
• POST /api/register — אימייל + סיסמה (bcrypt). יוצר קובץ משתמש.
• POST /api/login — מחזיר JWT.
• GET /api/vault — מחזיר ciphertext (או ריק).
• PUT /api/vault — שומר ciphertext + מטא (השרת לא מפענח).

דרישות
-------
• Node.js 18+
• משתנה סביבה JWT_SECRET — מחרוזת אקראית ארוכה (לפחות 16 תווים; מומלץ 32+).

הרצה מקומית (Windows / Mac / Linux)
------------------------------------
1. פתחו טרמינל בתיקייה vault-server
2. npm install
3. העתיקו .env.example ל-.env וערכו JWT_SECRET=
4. npm start
   ברירת מחדל: http://127.0.0.1:3847
5. בדפדפן או ב-curl: GET http://127.0.0.1:3847/api/health  → {"ok":true,...}

פריסה לשרת (VPS / ענן)
-----------------------
1. העלו את תיקיית vault-server לשרת (ללא תיקיית data/ מקומית אם יש — תיווצר מחדש).
2. הגדירו NODE_ENV=production, JWT_SECRET, PORT (אופציונלי), DATA_DIR (אופציונלי — נתיב לקבצי משתמשים).
3. CORS_ORIGIN: בפרודקשן מומלץ כתובת מדויקת של האתר שבו רצה האפליקציה (לא כוכבית).
4. הפעילו מאחורי HTTPS (Nginx / Caddy / Traefik) עם reverse proxy ל-localhost:PORT.
5. בלקוח (הגדרות האפליקציה) הזינו את כתובת הבסיס של השרת (https://... בלי סלאש בסוף).

גיבוי השרת
-----------
כל הקבצים ב-data/ הם «כספות» מוצפנות — גבו את התיקייה באופן קבוע.

אבטחה
------
• לעולם לא לשתף JWT_SECRET.
• אם JWT נגנב — תוקף 90 יום; עדיין אין פענוח בלי סיסמת האפליקציה המקומית, אבל תוקף יכול לאפשר דריסה של הענן — לכן HTTPS חובה.
