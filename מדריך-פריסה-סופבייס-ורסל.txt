================================================================================
מדריך פריסה מלא — ניהול משפחתי + Supabase + Vercel + דומיין
================================================================================
קראו לפי הסדר. אחרי כל צעד יש שורת "בדיקה" — אל תדלגו.

מילון מילה אחת:
• Supabase = מסד נתונים + התחברות משתמשים בענן (יש לכם חשבון).
• Vercel = מקום שמריץ את דף האפליקציה שלכם (יש לכם חשבון).
• GitHub = מקום לקוד (יש לכם חשבון).
• דומיין = הכתובת היפה של האתר (יש לכם).

מה נשמר בסופבייס?
רק מחרוזת מוצפנת (ciphertext). אף אחד בלי סיסמת האפליקציה שלכם לא פותח את התוכן.


חלק א — הכנת Supabase (מסד + הרשאות)
--------------------------------------------------------------------------------
צעד א1: היכנסו ל-https://supabase.com והתחברו.

צעד א2: לחצו "New project". בחרו ארגון, שם פרויקט, סיסמת מסד (שמירה — לא
        נחוצה לאפליקציה שלנו ישירות). אזור (Region) — אירופה קרוב לישראל
        (למשל Frankfurt) מתאים לרוב.

בדיקה א2: בלוח הבקרה של הפרויקט אתם רואים "Project Settings".

צעד א3: כבו אישור מייל להרשמה (מומלץ לשימוש משפחתי פשוט):
        Project Settings → Authentication → Providers → Email →
        כבו "Confirm email" (או השאירו דלוק ואז תצטרכו לאשר מייל אחרי הרשמה).

בדיקה א3: שמרתם מה שבחרתם בראש.

צעד א4: פתחו Project Settings → API. העתיקו לפניכם לקובץ טקסט זמני:
        • Project URL  (נראה כמו https://xxxx.supabase.co)
        • anon public key (מחרוזת ארוכה)

בדיקה א4: שתי השורות הועתקו במלואן.

צעד א5: בתפריט שמאל: SQL → New query. פתחו בקומפיוטר את הקובץ:
        supabase/schema.sql מהפרויקט, העתיקו את כל התוכן, הדביקו ב-SQL Editor,
        לחצו Run.

בדיקה א5: הודעת הצלחה ללא שגיאות. ב-Table Editor אמורה להופיע טבלה
        user_vaults (אולי ריקה).


חלק ב — GitHub + קוד
--------------------------------------------------------------------------------
צעד ב1: ודאו שכל הקבצים מהפרויקט (כולל supabase/, vercel.json, package.json)
        נמצאים בתיקייה אחת.

צעד ב2: צרו ריפו חדש ב-GitHub (או השתמשו בקיים). העלו את התיקייה
        (דרך GitHub Desktop / git push — איך שנוח לכם).

בדיקה ב2: בדפדפן, בעמוד הריפו ב-GitHub, אתם רואים את הקבצים:
        build-standalone.mjs, index.html, vercel.json, supabase/schema.sql


חלק ג — Vercel + דומיין
--------------------------------------------------------------------------------
צעד ג1: היכנסו ל-https://vercel.com והתחברו (אפשר "Import" עם חשבון GitHub).

צעד ג2: Add New → Project → בחרו את הריפו → Deploy.
        Vercel יזהה buildCommand ו-outputDirectory מ-vercel.json.

בדיקה ג2: ה-Deploy מסתיים בירוק. מקבלים כתובת כמו https://something.vercel.app

צעד ג3: פתחו את הכתובת בדפדפן — האפליקציה נטענת (מסך נעילה / הקמה).

בדיקה ג3: האתר נפתח ב-HTTPS (מנעול ירוק ליד הכתובת).

צעד ג4 (דומיין): ב-Vercel → Project → Settings → Domains → הוסיפו את
        הדומיין שלכם. עקבו אחרי ההוראות ל-DNS אצל רשם הדומיינים (רשומות
        A או CNAME לפי מה ש-Vercel מציג).

בדיקה ג4: https://הדומיין-שלכם פותח את אותה אפליקציה.


חלק ד — חיבור האפליקציה ל-Supabase (בממשק)
--------------------------------------------------------------------------------
צעד ד1: פתחו את האפליקציה מהכתובת הסופית (Vercel או הדומיין).

צעד ד2: התחברו עם סיסמת האפליקציה (או הקימו משתמש חדש).

צעד ד3: עברו להגדרות → כרטיס "סנכרון ענן (E2E) עם Supabase".

צעד ד4: מלאו:
        • כתובת Supabase (Project URL)
        • מפתח anon (המפתח הציבורי מה-API)
        • אימייל + סיסמה לסופבייס (לפחות 6 תווים לסיסמה)
        לחצו "שמור חיבור לסופבייס".

צעד ד5: לחצו "הרשמה בסופבייס" (פעם ראשונה) או "התחברות לסופבייס".

בדיקה ד5: הסטטוס משתנה ל"מחובר". אם מבקשים אישור מייל — חזרו לצעד א3.

צעד ד6: לחצו "דחיפה לענן (מוצפן)".

בדיקה ד6: הודעת הצלחה. ב-Supabase → Table Editor → user_vaults נראית שורה
        עם ciphertext ארוך (לא קריא).

צעד ד7: ממכשיר/דפדפן אחר: אותה כתובת אתר, אותם שדות סופבייס, אותו אימייל
        וסיסמת סופבייס, "התחברות", ואז "משיכה מהענן" (לאשר את האזהרה).
        סיסמת האפליקציה חייבת להיות זהה למכשיר שהעלה.

בדיקה ד7: הנתונים מופיעים במכשיר השני.


חלק ה — אם משהו נכשל
--------------------------------------------------------------------------------
• "supabase לא זמין" — טעינת הספרייה נכשלה; רענון קשיח (Ctrl+F5).
• שגיאת הרשמה — כבו Confirm email או אשרו מייל.
• משיכה לא מפענחת — סיסמת האפליקציה שונה בין המכשירים.
• RLS / permission — הריצו שוב את schema.sql וודאו שאין שגיאות.
